强化学习

随机算法，最短路径优先，最低负载优先

NFV（Network Function Virtualization）网路功能虚拟化，软硬件解耦以及功能抽象，使得网络设备不在依赖专用设备。自动部署，弹性伸缩，故障隔离，自愈等等。

VNF（Virtualized Network Function）虚拟网络功能，以一定顺序部署在服务器节点上，构成一条可以处理特定网络业务的网络功能序列（SFC，Service Function Chain，服务功能链）

SFC（Service Function Chain）服务功能链，作为网络与服务的重要承载形式，通过合理编排虚拟化的网络服务功能从而构建完整的端到端网络服务。

目标在于对相关指标（时延，负载，资源利用率，平衡负载）进行优化，从而优化SFC构建过程，提高构建效果。

优化问题

  1. 针对网络资源优化，主要是CPU，内存等节点资源和以带宽为代表的链路资源。目标在于最小化资源消耗和最大化资源利用率。
  2. 针对网络性能优化，主要包括时延，负载，吞吐量，服务链构建时间和构建成功率。目标在于服务功能链自身性能优化和面向的业务性能优化。自身性能优化包括降低服务链构建时间，提高成功率等，业务性能优化包括降低时延，提高网络吞吐率。

Q-learning算法

创建一个表格Q-Table，通过这个表格，可以为每一个状态（state）上进行的每一个动作（action）计算出最大的未来奖励（reward）的期望。

Q-Table每一列代表动作，每一行代表状态，每个单元格的值代表给定状态和相应动作的最大未来奖励期望。

每个Q-Table的分数将代表在给定最佳策略的状态下采取相应动作获得的最大未来期望。

可以看作备忘录型的动态规划

  • 为了学习到Q-table的每个值，使用Q-learning算法
    • 学习动作值函数（action value function）

Q-learning算法：动作值函数

Input：状态和动作

output：在该状态下执行该动作的未来奖励期望

在探索环境之前，Q-Table会给出相同的任意的设定值，随着持续探索，Q-Table会通过迭代使用Bellmen方程更新Q(s,a)给出越来越好的近似。

Q-learning伪代码：

  1. 初始化Q-value（Q(s,a)），m列（m动作数），n行（n状态数）的Q-Table，并进行初始化。

  2. 在整个生命周期中，3到5步会一直被重复，直到达到了最大训练次数或手动中止。

  3. 选取一个动作，在基于当前Q值估计的出的状态s下选择一个动作a

    如果开始每个Q值都等于0，使用ε（epsilon）贪婪策略：

    • 指定一个探索速率ε，一开始设定为1，意味着需要通过随机选择动作进行大量探索。
    • 生成一个随机数，如果这个数 ≥ ε，将进行利用已经知道的信息选择动作，否则继续进行探索。
    • 在刚开始训练Q函数时，必须有一个大的ε，随着探索逐渐减少ε。

  4. 采用动作a并观察输出的状态s’和奖励r。

  5. 使用Bellmen方程去更新Q(s，a)：

可以将学习率看作网络有多块的抛弃旧值、生成新值的度量。学习率为1代表信的估计值会成为新的Q值，完全抛弃旧值。γ越接近1代表越看重后续状态的价值，越接近0就越考虑当前的利益。

Q-learning主要优势是使用了时间差分法（TD）（融合了蒙特卡洛和动态规划）能够进行离线学习，使用bellman方程可以对马尔可夫过程求解最优策略。

动态规划法：需要一个完全已知的环境，需要状态之间的转换概率，并且状态值函数V(s)的估计是自举（bootstrapping）即当前状态值函数的更新依赖于已知的其他状态值函数（bellman方程用后续状态）。

蒙特卡洛法：需要执行一个完整的episode才可以进行估计。

值函数

分为状态值函数和动作值函数，分别用来表示状态和状态下执行某动作的好坏程度。

每个状态的值不仅有当前状态决定，还要由后面的状态决定，所以状态的累计奖励求期望就可得出当前s的状态值函数V(s)。
